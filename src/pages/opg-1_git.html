Versionsstyring med Git og Gruppearbejde

Præsenter Git fra opsætning til generel brug. Uddyb processerne initialisering, staging og comitting. Branching og merging skal også dækkes. Inkluder cloning, pushing, pulling samt brug af et fælles serverbaseret repository.

Introduktion til Git

Git er et versionskontrolsystem, der bruges til at spore ændringer i kode og samarbejde mellem udviklere. Det giver dig mulighed for at arbejde sikkert på dit projekt uden at miste historik over dine ændringer.

Hvordan fungerer Git?

Initialisering af et Git Repository

Hvordan gør man?

Start med at køre git init i roden af en lokalt projektmappe for at oprette et nyt repository.

Det svarer til at du står med en tom pluksedel som du skal udfylde med en liste af varer.

I vores tilfæde er det ikke varer, men kode.

Staging (git add)

Hvordan gør man?

Med git add markerer du ændringer, der skal med i den næste commit. Disse ændringer placeres i "staging area",

Det svarer til at man begynder at skrive varer på pluksedlen. Dette kan være:

  • Nye kodefiler og mapper, som bliver tilføjet til listen.
  • Ændringer i eksisterende kodefiler eller placeringen af dem.
  • Sletning af kodefiler eller mapper, der ikke længere skal bruges.
