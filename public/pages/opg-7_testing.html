Hvad er Testning?

Testning er en proces i softwareudvikling, der sikrer, at programmet opfører sig som forventet, og at eventuelle fejl eller uforudsete opførsel identificeres og rettes. Det er en afgørende del af softwareudviklingscyklussen, der hjælper med at forbedre pålideligheden og kvaliteten af software.

Formålet med testning er:

  • At opdage fejl tidligt: Jo tidligere fejl opdages, desto billigere og hurtigere er det at rette dem.
  • At sikre korrekt funktionalitet: Test sikrer, at funktioner lever op til kravene.
  • At forhindre regression: Test forhindrer tidligere fungerende kode i at bryde efter ændringer.

Testmetoder

Manuel Testning

Tester udføres manuelt af udviklere eller testere. Velegnet til brugeroplevelser og visuelle elementer.

Ulempe: Tidskrævende og svært at gentage konsekvent.

Eksempel: En tester åbner en webapplikation, klikker på knapper og tjekker resultatet.

Automatiseret Testning

Bruger scripts og værktøjer til hurtigt at teste applikationer. Velegnet til gentagne tests og regressionstest.

Fordel: Hurtigere og mere pålideligt end manuel testning.

Eksempel: Et script tester automatisk, om en "tilføj til kurv"-funktion fungerer korrekt.

Testniveauer

1. Enhedstest (Unit Test)

Tester individuelle moduler eller funktioner i isolation. Hjælper med at sikre, at hver del fungerer korrekt.

Eksempel: Test af en funktion, der beregner summen af to tal.

2. Integrationstest

Tester, hvordan forskellige moduler arbejder sammen.

Eksempel: En integrationstest sikrer, at en databaseforespørgsel returnerer korrekte data til en API.

3. Systemtest

Tester hele systemet som en helhed. Fokus på funktionalitet og ydeevne.

Eksempel: Systemtesten sikrer, at en e-handelsplatform korrekt behandler ordrer.

4. Accepttest

Validerer, at systemet opfylder krav og forventninger.

Eksempel: Bruges til at bekræfte, at produktet er klar til lancering.

Hvad er Unittest og Jest?

Unittest er en testmetode, der fokuserer på at teste mindre, isolerede dele af koden. Jest er et populært JavaScript-testframework, der gør testning enkel og effektiv.

Eksempel på Jest

Funktion

const add = (a, b) => a + b;

Test

test("add function should return the sum of two numbers", () => {
    expect(add(2, 3)).toBe(5);
    expect(add(-1, 1)).toBe(0);
});

Konklusion

Testning er afgørende for softwareudvikling. Manuel testning kan bruges til små projekter, men automatiseret testning er essentiel for at sikre korrekt og konsistent funktionalitet. Jest er et fremragende værktøj til unittests og gør processen enkel og effektiv.

Unittest og Jest

Unittest er en testmetode, der fokuserer på at teste mindre, isolerede dele af koden, som for eksempel funktioner eller klasser. Et populært værktøj til unittest er Jest, som er et JavaScript-testframework.

Hvad er Jest?

Jest er et kraftfuldt værktøj til at skrive unittests for JavaScript- og TypeScript-projekter. Det tilbyder funktioner som mockning, testdækning og asynkron testning.

Opsætning af Jest

  1. Installer Jest: npm install --save-dev jest
  2. Tilføj en testscript i package.json: 
"scripts": {
    "test": "jest"
}

Eksempel på Unittest

Her er et eksempel på en funktion og dens unittest:

// Funktion
const add = (a, b) => a + b;

// Test
test('add function should return the sum of two numbers', () => {
    expect(add(2, 3)).toBe(5);
    expect(add(-1, 1)).toBe(0);
});

Mockning med Jest

Jest gør det nemt at mocke funktioner, for eksempel API-kald:

const fetchData = jest.fn().mockResolvedValue({ data: 'Hello, World!' });

// Test
test('fetchData should return mocked data', async () => {
    const data = await fetchData();
    expect(data).toEqual({ data: 'Hello, World!' });
});

Server Test Setup

package.json

{
  "devDependencies": {
    "jest": "^29.7.0",
    "supertest": "^7.0.0"
  },
  "dependencies": {
    "express": "^4.21.1"
  },
  "scripts": {
    "test": "jest"
  }
}

Denne konfiguration gør det muligt at bruge jest til tests og supertest til HTTP-anmodninger.

Testkode

const request = require("supertest");
const app = require("../server"); // Din app
let server;

beforeAll((done) => {
  server = app.listen(3000, done);
});

afterAll((done) => {
  server.close(done);
});

describe("Test server", () => {
  it("skal returnere status 200 for hjemmet (index)", async () => {
    const response = await request(app).get("/");
    expect(response.status).toBe(200);
  });

  it("skal returnere status 404 for ikke-eksisterende ruter", async () => {
    const response = await request(app).get("/non-existent-page");
    expect(response.status).toBe(404);
  });
});

Denne test sikrer, at din server reagerer korrekt på kendte og ukendte ruter.

  • Sikrer bedre samarbejde i udviklingsteams.
