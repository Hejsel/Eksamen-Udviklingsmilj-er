Unittest & Jest

Hvad er Testning?

Testning er en afgørende del af softwareudvikling. Formålet er at sikre, at koden fungerer som forventet, og at fejl opdages og rettes tidligt i udviklingsprocessen.

Testning kan opdeles i flere niveauer og metoder, der hver især tjener specifikke formål. Her dykker vi ned i nogle af de mest anvendte.

Testmetoder og Testniveauer

Testmetoder

  • Manuel Testning: Tester udføres manuelt af udviklere eller testere for at finde fejl.
  • Automatiseret Testning: Skripter og værktøjer bruges til at teste applikationer hurtigt og effektivt.

Testniveauer

  • Enhedstest (Unit Test): Tester individuelle moduler eller funktioner.
  • Integrationstest: Tester hvordan forskellige moduler arbejder sammen.
  • Systemtest: Tester hele systemet som en helhed.
  • Accepttest: Validerer, at systemet opfylder krav og forventninger.

Unittest og Jest

Unittest er en testmetode, der fokuserer på at teste mindre, isolerede dele af koden, som for eksempel funktioner eller klasser. Et populært værktøj til unittest er Jest, som er et JavaScript-testframework.

Hvad er Jest?

Jest er et kraftfuldt værktøj til at skrive unittests for JavaScript- og TypeScript-projekter. Det tilbyder funktioner som mockning, testdækning og asynkron testning.

Opsætning af Jest

  1. Installer Jest: npm install --save-dev jest
  2. Tilføj en testscript i package.json: 
"scripts": {
    "test": "jest"
}

Eksempel på Unittest

Her er et eksempel på en funktion og dens unittest:

// Funktion
const add = (a, b) => a + b;

// Test
test('add function should return the sum of two numbers', () => {
    expect(add(2, 3)).toBe(5);
    expect(add(-1, 1)).toBe(0);
});

Mockning med Jest

Jest gør det nemt at mocke funktioner, for eksempel API-kald:

const fetchData = jest.fn().mockResolvedValue({ data: 'Hello, World!' });

// Test
test('fetchData should return mocked data', async () => {
    const data = await fetchData();
    expect(data).toEqual({ data: 'Hello, World!' });
});

Fordele ved Testning

  • Reducerer risikoen for fejl i produktionen.
  • Øger vedligeholdelsesmulighederne for koden.
  • Sikrer bedre samarbejde i udviklingsteams.

TypeScript: Stærkere Typing i JavaScript

Fra opsætning til komplekse typer

Hvad er TypeScript?

TypeScript er et superset af JavaScript, der tilføjer statisk typing. Det hjælper med at opdage fejl tidligere og gør koden mere læsbar og vedligeholdelig.

Sammenligning: JavaScript vs. TypeScript

JavaScript

const greet = (name) => {
    return `Hello, ${name}!`;
};

TypeScript

const greet = (name: string): string => {
    return `Hello, ${name}!`;
};

Type Inference

TypeScript kan automatisk udlede typer baseret på tildelte værdier:

let number = 5; // TypeScript ved, at dette er en 'number'
let greeting = "Hej"; // TypeScript ved, at dette er en 'string'

Komplekse Typer

TypeScript gør det muligt at definere avancerede typer, f.eks. interfaces og union-typer: 

interface User {
    id: number;
    name: string;
    email?: string; // Valgfri
}

const users: User[] = [
    { id: 1, name: "Emil" },
    { id: 2, name: "Sofie", email: "sofie@example.com" }
];

const getUserName = (user: User): string => user.name;

Opsætning og Kompilering

  1. Installer TypeScript: npm install -g typescript
  2. Opret en tsconfig.json fil:
{
    "compilerOptions": {
        "target": "ES6",
        "module": "commonjs",
        "strict": true
    }
}

Kompilér TypeScript med tsc kommandoen.
